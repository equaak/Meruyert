## Шаг 1. Создать папку проекта через Claude

1. Откройте VS Code.
2. Установите расширение **Claude Code** (Ctrl+Shift+X → найти «Claude Code» от Anthropic → Install). Залогиньтесь в claude.ai.
3. Откройте чат Claude (иконка Claude в левой панели, либо Ctrl+Shift+P → «Claude: Open Chat»).
4. Отправьте промпт ниже. После того как Claude скажет «готово» — нажмите File → Open Folder в VS Code и выберите `C:\Projects\kaspi-dashboard`.

```
Я разворачиваю Kaspi Dashboard на своём Windows-ноутбуке. Первое действие: создай папку C:\Projects\kaspi-dashboard (если C:\Projects не существует — создай и её). Когда папка создана — скажи мне открыть её через File → Open Folder в VS Code, чтобы дальше ты работал внутри неё.
```

## Шаг 2. Установить Node.js + Git + CLI-инструменты

Один промпт ставит всё необходимое через winget и npm.

```
Установи на мою Windows-машину:

1. Node.js 20 LTS — winget install OpenJS.NodeJS.LTS
2. Git for Windows — winget install Git.Git (если ещё не стоит)
3. GitHub CLI — winget install GitHub.cli

После установки Node.js закрой и заново открой терминал чтобы PATH обновился, затем поставь Vercel CLI: npm install -g vercel

Проверь: node --version (должно показать v20.x), git --version, gh --version, vercel --version. Если что-то не встало — расскажи мне человеческим языком.
```

## Шаг 5. Подключить токены Kaspi (всё локально, ничего никому не пересылаем)

Все 5 значений вы добавляете сами через Claude. Ничего не отправляется по почте/мессенджеру — токен живёт только на вашем ноутбуке и (на Шаге 8) в вашем Vercel.

### Откуда взять каждое значение

**1. `KASPI_TOKEN`** — ваш персональный токен.
- Открыть [kaspi.kz/merchantcabinet/](https://kaspi.kz/merchantcabinet/) → ваш логин магазина → **Настройки → API**
- Нажать **Сгенерировать токен** (или скопировать существующий)
- Это длинная base64-строка, заканчивается на `=`

**2. `KASPI_MERCHANT_ID`** — UID вашего магазина.
- На той же странице **Настройки → API** — отображается как «Merchant UID» или «Идентификатор магазина»
- Обычно 8 цифр, типа `16974112`

**3. `KASPI_MERCHANT_NAME`** — название магазина для отображения в шапке дашборда.
- Любая строка по вашему вкусу: `Beauty Store · Kaspi Marketplace`, `Магазин ABC`, и т.д.
- Можно русским, можно с эмодзи

**4. `NICHE_USER`** — логин для входа в **сам дашборд** (Basic Auth в браузере).
- Придумываете сами. Например, `merchant` или `admin`

**5. `NICHE_PASS`** — пароль для входа в дашборд.
- Придумываете сами или пусть Claude сгенерирует надёжный (16+ символов, буквы+цифры+спецсимволы)

### Промпт Claude

Отправьте промпт ниже, **заполнив все 5 значений** перед отправкой (никаких плейсхолдеров не оставляем):

```
Создай файл .env.local в корне моего проекта C:\Projects\kaspi-dashboard с этими переменными (формат KEY=значение, каждая на отдельной строке):

KASPI_API_BASE=https://kaspi.kz/shop/api/v2
KASPI_TOKEN=ВСТАВЬТЕ_СЮДА_ВАШ_ТОКЕН_ИЗ_KASPI_CABINET
KASPI_MERCHANT_ID=ВСТАВЬТЕ_СЮДА_ВАШ_MERCHANT_UID
KASPI_MERCHANT_NAME=ВСТАВЬТЕ_СЮДА_НАЗВАНИЕ_ВАШЕГО_МАГАЗИНА
NICHE_USER=ВСТАВЬТЕ_СЮДА_ЛОГИН_ДЛЯ_ВХОДА_В_ДАШБОРД
NICHE_PASS=ВСТАВЬТЕ_СЮДА_ПАРОЛЬ_ДЛЯ_ВХОДА_В_ДАШБОРД

Проверь что .env.local попадает в .gitignore (не должен уйти в репо). После создания — выведи мне список добавленных ИМЁН переменных (без значений), чтобы я убедился что все 6 на месте.
```

Если хотите чтобы Claude **сам сгенерировал NICHE_PASS** — добавьте в конце промпта: «Для NICHE_PASS — сгенерируй сам безопасный 20-символьный пароль и покажи мне, я его сохраню».

```

Должна появиться форма Basic Auth — введи мои NICHE_USER / NICHE_PASS из .env.local. После входа должна открыться главная страница /kaspi с реальными данными моего магазина.

Если 1) форма Basic Auth не появляется или 2) после входа «нет данных» — открой /settings и расскажи мне, что показывает блок «Подключение к Kaspi Marketplace» (зелёное / красное, и текст ошибки).
```

## Шаг 8. Залить env-переменные в Vercel

Есть два пути — выбери удобный.

### Путь А (через CLI, быстрее)

```
Залей все 6 переменных из моего локального .env.local в мой Vercel-проект kaspi-dashboard. Для каждой переменной запусти:

echo "значение" | vercel env add ИМЯ_ПЕРЕМЕННОЙ production preview development

КРОМЕ KASPI_MERCHANT_NAME — её НЕ заливай через CLI (там кириллица, у Vercel CLI с этим баг). Скажи мне добавить её через UI вручную.

В конце выведи vercel env ls и дай ссылку на Settings → Environment Variables моего проекта.
```

После CLI зайди в Vercel UI → проект → Settings → Environment Variables → Add New:
- **Key:** `KASPI_MERCHANT_NAME`
- **Value:** название магазина (русскими буквами можно)
- **Environments:** Production, Preview, Development
- **Save**

### Путь Б (всё через UI, надёжнее для кириллицы)

Vercel Dashboard → ваш проект `kaspi-dashboard` → Settings → Environment Variables → Add New. Добавьте по одной все 6 переменных, для каждой выбрав все три окружения.

## Шаг 9. Redeploy чтобы прод подхватил переменные

```
Сделай Redeploy последнего деплоя на Vercel чтобы прод подхватил новые env-переменные. После деплоя — открой прод-URL и убедись, что Basic Auth работает (логин/пароль из NICHE_USER/NICHE_PASS) и после входа /kaspi показывает реальные данные. Если что-то не так — расскажи, что видишь на /settings.
```

## Дальше — что можно доработать

Скажите Claude словами, что хотите видеть. Примеры запросов которые точно работают:

- *«Добавь на /kaspi-zakazy фильтр по способу оплаты»*
- *«Покажи в /kaspi-klienty средний интервал между заказами для клиентов с 2+ заказами»*
- *«Сделай страницу /kaspi-vyrucka с динамикой выручки по неделям с прогнозом»*

Claude умеет читать код, тестировать на localhost, коммитить и пушить в ваш репо. Vercel автоматически выкатит обновление.
